# 🎯 智能五子棋：高精度视觉与 AI 博弈中枢

欢迎使用本套“工业级”实体五子棋视觉与 AI 决策系统！
本系统专为自动化对弈、桌面机器人开发、高校毕业设计（机器视觉/人工智能方向）等场景打造。它彻底摒弃了传统死板的 OpenCV 全局调参，采用 **“动态局部差分+GUI实时调参”**，并内置了纯 Python 编写的 **“带有记忆进化能力的 Minimax AI 引擎”**。

无需昂贵的特定机械臂，只要有普通的 USB 摄像头和电脑，您就能瞬间拥有一个能看懂物理局势、会自我进化、并带有赛博朋克风 UI 的“外挂级”大脑！

## ✨ 核心产品壁垒 (为什么选我们？)

* 🖥️ **赛博朋克 UI 中枢**：采用 PyQt5 打造的极客暗黑风控制台。内置 3D 拟真棋盘渲染，AI 落子位置带有极具科幻感的“呼吸光晕”锁定特效。
* 🧠 **大师级 AI 引擎**：内置极小极大值 (Minimax) 与 Alpha-Beta 剪枝算法。不仅走一步看三步，还带有“最快击杀惩罚机制”，能两步赢绝不拖到第三步！
* 🧬 **AI 基因记忆进化系统 (核心亮点)**：系统内置本地经验记忆库 (`ai_experience.json`)。AI 每次深度推演后的“最优解”都会被生成独一无二的 MD5 盘面指纹并永久保存。同样的残局，第二次遇到只需 0.001 秒即可调出绝杀坐标！玩的次数越多，系统越强！
* 🛡️ **光照免疫与防边缘裁切**：特有的局部自适应检测算法，并带有 OpenCV 实时滑块调参（可独立调节黑白子敏感度）。底层引入 35 像素“边缘安全区”，即使棋子下在最角落也绝对不会“隐身”。
* ⚖️ **智能盘面裁判系统**：带有防呆与作弊检测机制。允许玩家有 1 颗棋子的拿放误差（动态判定先后手），但如果相差 2 颗及以上，UI 界面将直接红牌警告，拒绝提供算力推演！

本系统为纯软解耦架构，无需复杂的硬件依赖：
* **💻 电脑**：普通轻薄本或台式机（Windows / macOS / Linux），能跑 Python 即可。

* **🦾 支架（推荐）**：简易的桌面悬臂支架或手机三脚架，将摄像头架在物理棋盘上方。
* **🏁 物理棋盘与棋子**：标准的实体五子棋盘（系统默认配置为 15x15，代码开头的 `BOARD_SIZE` 可随时修改）与普通黑白棋子。

## 📦 1. 源码包文件结构

解压后，您的工程目录下应包含以下核心资产：
1. `main.py`：核心控制中枢（包含 OpenCV 视觉线程、PyQt5 UI 渲染、主控逻辑）。
2. `ai_engine.py`：博弈大脑（包含 Minimax 算法、盘面裁判、记忆进化机制）。
3. `calibration_tool.py`：傻瓜式视觉标定外挂（解决手工量取透视坐标的痛点）。
4. `requirements.txt`：Python 环境依赖清单。

## 🛠️ 2. 环境准备与极简部署

本系统无臃肿的深度学习框架（不依赖 PyTorch/TensorFlow），普通电脑也能跑到满帧率。

   ```bash

   ```

*(注：核心依赖仅需 `opencv-python`, `numpy`, `PyQt5`)*

## 🚀 3. 傻瓜式运行指南 (只需三步)

### 第一步：运行标定外挂，获取专属物理坐标

不要再去手工量坐标了！请保持物理棋盘和摄像头位置固定，运行：

```bash
python calibration_tool.py

```

在弹出的实时画面中，**用鼠标依次点击物理棋盘最外圈的四个角（严格按照：左上 ➡️ 右上 ➡️ 右下 ➡️ 左下）**。点完后，控制台会自动生成一段专属的 `SRC_POINTS` 代码。

### 第二步：将坐标填入主程序

复制生成的 `SRC_POINTS` 代码，使用代码编辑器打开 `main.py`，在开头的 **`【全局配置区】`** 替换掉原有的坐标。
*(您也可以在这里修改 `CAMERA_ID` 切换摄像头，或修改 `BOARD_SIZE` 切换棋盘大小)*。

### 第三步：启动天眼智控中枢！

保存后，直接运行主程序：

```bash
python main.py

```

系统将同时弹出两个窗口：

1. **上帝视角调试窗 (God's Eye)**：您可以通过顶部的滑块，实时微调黑白子的光线敏感度，直观看到识别红圈。
2. **极客中枢控制台**：点击“请求 AI 决策”，AI 将进行深度推演或调用基因库。随后棋盘上会出现极具科技感的闪烁光晕，等您在物理世界中放置好棋子后，系统会自动确认并进行下一步！

## ❓ 4. 常见问题排查 (Q&A)

**Q1：运行报错 `cv2.error`，或弹出的画面是灰色的/黑屏？**

> **解答**：这 100% 是摄像头编号不对或被占用。请尝试把 `main.py` 中的 `CAMERA_ID` 从 1 改成 0，或检查视频会议软件是否正在占用摄像头。

**Q2：画面里的红十字/识别圈没有对准物理棋盘的交叉点？**

> **解答**：第一步的标定没点准。请重新运行 `calibration_tool.py`，仔细点击棋盘**最外围边线**的四个交点（带有一点点留白没关系，底层算法会自动处理 35 像素的边缘裁切保护），并重新替换坐标。

**Q3：点击请求 AI 决策，界面变成了红色警告？**

> **解答**：触发了盘面裁判机制！请检查物理棋盘：是否不小心碰乱了棋子？是否连下了两颗黑子？只要修正物理棋盘，保持黑白子数量一致（或相差1颗），再次点击即可恢复推演。

**Q4：为什么有时候 AI 秒出结果，有时候要思考 1-2 秒？**

> **解答**：恭喜您触发了【AI 基因记忆库】！如果当前残局在历史对弈中出现过，AI 会直接从硬盘的 `ai_experience.json` 中以 0.001 秒读取最优解；如果是全新盘面，AI 则会启动深度的极小极大值算法进行推演。
